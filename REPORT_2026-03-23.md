## 1. Tech Stack Decisions

### Why These Choices

| Technology | Rationale |
|-----------|-----------|
| **React 19** | Latest stable release. Strict purity rules enforced by linter catch subtle bugs early. |
| **Vite 8** | Fastest dev server and build tool available. 600ms production builds. |
| **HTML5 Canvas** | Chosen over WebGL for simplicity. 60fps achievable for this complexity level. Web Audio API for procedural sound (no audio files needed). |
| **Vanilla CSS** | No Tailwind, no CSS-in-JS. Single consolidated `index.css` for zero runtime overhead and full control over animations. |
| **ESLint 9 + React Hooks plugin** | Enforces React 19 purity rules (no `Math.random()` in render, no ref access during render). Catches bugs at build time. |

| **Knip** | Unused dependency/export detection | Cleaned 3 unused exports, 1 false positive configured |
| **ESLint** | Lint + React 19 rules | Zero errors, zero warnings on final build |

## 3. Bugs & Issues

### CRITICAL: Curve Plateau Bug (Partially Fixed)

**Status:** Fixed Y-axis scale locked, but curve still visually stalls at high multipliers.

**What was done:**
- Changed rate constant from `k = ln(crash) / 20` to `k = ln(crash) / 12` (33% faster)
- Locked Y-axis to `crashMultiplier * 1.15` on flight start (no more dynamic rescaling)

**What's still wrong:**
- The exponential curve is asymptotic — it approaches the crash multiplier but never reaches it
- At 20x crash: after 10s the multiplier is ~16x, after 15s ~18x, last stretch visually stalls
- The curve needs a "linear kick" in the final stretch to visibly hit the crash point

**Planned fix:** Add a blend factor in the last 10% of estimated flight time that linearly interpolates toward the crash multiplier instead of asymptotically approaching it.

### CRITICAL: "Flew Away" Not Triggering Properly

**Status:** Text component exists, but the multiplier continues climbing past the crash point in some cases.

**What's happening:**
- The `getMultiplierAtTime` function returns `Math.min(multiplier, crashMultiplier)` which should cap it
- But the renderer receives `state.multiplier` from the engine, which is set via `getMultiplierAtTime`
- The engine's `_updateFlying` checks `if (multiplier >= crashMultiplier)` to trigger `_crash()`
- There's a timing gap where the multiplier can briefly exceed the crash point before the state transition completes

**Planned fix:** Ensure the multiplier display always shows `crashMultiplier` when state is CRASHED, and add a guard in the tick handler to force-cap the displayed multiplier.

### HIGH: Plane Animation Stalling

**Status:** Plane bob uses `easeInOutQuad` on a sine wave, but the plane's vertical position can appear static when the curve is plateauing.

**Root cause:** This is a side effect of the curve plateau bug above. The plane follows the curve, so if the curve stalls, the plane stalls. Fixing the curve plateau will fix this.

### MEDIUM: Background Music Not Implemented

**Status:** Sound effects (SFX) are complete, but no ambient background music track.

**What exists:** The `SoundEngine` class has the architecture for it (Web Audio API gain nodes), but no ambient loop or background track is generated.

**Planned fix:** Add a low-frequency ambient drone (60-80Hz sine + filtered noise) that plays continuously during gameplay. Can be toggled via the Settings menu "Music" toggle.

### MEDIUM: No Start/Lobby Menu

**Status:** The in-game hamburger settings menu exists, but there's no pre-game start screen or lobby menu.

**What exists:** The game starts immediately on page load. The LOBBY state shows the sponsor banner but no "Start Game" or "Join" button.

**Planned fix:** Add a start screen overlay that appears before the game loop begins. Include game title, sponsor branding, and a "Start Playing" button that initializes the engine.

### LOW: BetTable Top Tab Data is Static

**Status:** Top tab shows rich player cards, but data is generated once per mount and doesn't refresh.

**Root cause:** `useState(() => generateBets())` runs once. The `key={roundNumber}` forces remount on new rounds, which regenerates data, but only for the All Bets tab.

### LOW: Player Count is Not Truly Dynamic

**Status:** Player count fluctuates every 5 seconds (+/- 10), but it's cosmetic — not tied to any real player simulation.
