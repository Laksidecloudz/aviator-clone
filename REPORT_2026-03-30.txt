## 2. Bug Fix: Multiplier Sluggishness

**File:** `src/game/CrashAlgorithm.js`

### Problem

Multiplier felt sluggish at higher values because the formula
**decelerated** (started fast, slowed down as it approached the crash
point).

### Solution

Changed from decelerating to **accelerating** formula.

### Code Change

**Old Formula (Decelerating):**
`javascript export function getMultiplierAtTime(flightTimeMs, crashMultiplier) {   if (crashMultiplier <= 1.00) return 1.00;   const t = flightTimeMs / 1000;   const k = computeRateConstant(crashMultiplier);   return 1 + (crashMultiplier - 1) * (1 - Math.exp(-k * t)); }`

**New Formula (Accelerating):**
`javascript export function getMultiplierAtTime(flightTimeMs, crashMultiplier) {   if (crashMultiplier <= 1.00) return 1.00;   const t = flightTimeMs / 1000;   const k = computeRateConstant(crashMultiplier);   const normFactor = Math.exp(k * 8) - 1;   return 1 + (crashMultiplier - 1) * (Math.exp(k * t) - 1) / normFactor; }`

### Impact

-   Multiplier now accelerates throughout flight
-   Creates tension as it speeds toward crash point
-   Matches real Spribe Aviator feel
